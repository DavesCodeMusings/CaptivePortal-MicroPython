Captive WiFi Portal

Are there are existing microcontroller based captive portals? Yes. Some are even written in MicroPython. But this one may be unique in its use of asyncio. If nothing else, it gives some insight into an extremely minimalist DNS server and HTTP server.

This captive portal implementation uses the older method of hijacking DNS and HTTP requests to redirect to a portal.html page. Newer captive portals use a DHCP option to point client machines to a captive portal API. This can't be done in MicroPython, because there is no way to configure DHCP options for the access point. Still, it works reasonably well. (Sometimes you have to nudge it by attempting to open a site like http://example.com in a browser.)

The idea behind this project is not so much practical as it is microcontroller-based graffiti art. Construct a portal.html page with your message. Create an SSID that entices people to connect. Stash the microcontroller somewhere people will come across its SSID.

Code is at: https://github.com/DavesCodeMusings/CaptivePortal-MicroPython
